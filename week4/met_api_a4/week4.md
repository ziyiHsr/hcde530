# Week 4 Competency Claim and HCD Reflection

## Competency Claim

For this assignment, I used a real public API outside of class: The Metropolitan Museum of Art Collection API. I wrote a Python script that makes a request to the API, parses the JSON response, extracts multiple fields from each record, and saves the structured output to a CSV file. The script retrieves at least 50 records and runs without errors.

This work demonstrates that I can read API documentation, identify useful endpoints, make structured API calls, and transform JSON data into a readable format for further analysis. I also added inline comments to explain what the script is doing, what the API returns, and why I selected specific fields.

## HCD Reflection

This workflow is relevant to HCD because designers often need to work with structured data in order to support search, discovery, and interpretation in digital systems. In this case, the Met API makes artwork records available in a format that could support browsing interfaces, educational tools, or recommendation features.

For HCD work, the value is not only in retrieving data, but also in understanding how data structure shapes what users can access and how they interact with information. Fields such as title, artist name, date, department, and medium can help organize cultural content in ways that are more searchable, understandable, and meaningful.

This assignment also reminded me that working with API data requires more than writing code. It requires checking what the API actually returns, selecting fields intentionally, and making the output readable for human use. That kind of structured thinking is important in design practice, especially when building systems that connect people with large collections of information.
